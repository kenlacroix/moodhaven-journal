Era 1 October 2025 v0.1.0

Foundation

"The non-negotiables, from day one."

AES-256-GCM encryption with PBKDF2 (600,000 iterations). The backend never sees plaintext. TOTP and FIDO2 hardware keys via native Rust CTAP2 — no browser WebAuthn. A mood scale. A TipTap editor. WebDAV sync. Oura integration. The ground floor was load-bearing from the start.

  • AES-256-GCM encryption
  • PBKDF2 · 600k iterations
  • Mood 1–5 scale
  • TipTap rich-text editor
  • TOTP 2FA
  • FIDO2 hardware key (Rust)
  • 24-char recovery key
  • WebDAV cloud sync
  • Location & weather
  • Privacy modes per entry
  • Full-text search (Ctrl+K)
  • On This Day view
  • Oura Ring integration
  • Focus mode
Backend never sees plaintext — enforced on day one, not retrofitted.
Era 2 Nov–Dec 2025 v0.2.0–v0.3.2

Intelligence

"Making the journal talk back."

Analytics surfaced patterns the user couldn't see in individual entries. AI turned those patterns into prompts and insights — using only anonymised metadata, never journal text. The first-run wizard made onboarding coherent. Seven journal templates gave new writers a starting point.

  • Calendar heatmap
  • Mood trend chart
  • Streak tracking
  • Day-of-week patterns
  • AI insights (BYOK + Ollama)
  • Metadata-only AI (never journal text)
  • First-run setup wizard
  • Encrypted export · import
  • Factory reset
  • 7 journal templates
AI sends only anonymised metadata — mood scores, patterns, frequency. Journal text never leaves the device.
Era 3 Feb–Mar 2026 v0.4.0–v0.5.0

Era 4 March 2026 v0.6.0–v0.7.15

Connection

"Offline-first, device-aware."

The biggest era by commit count. Ed25519 device identity, mDNS discovery, PIN/QR pairing, and an AES-256-GCM TCP sync engine. Whisper.cpp speech-to-text via a local sidecar. A 3-layer formatting pipeline. Time capsules. The Wear OS companion. And a full rebrand to MoodHaven Journal.

  • Ed25519 device identity
  • mDNS peer discovery
  • PIN / QR pairing
  • AES-256-GCM peer sync
  • Port 44000–44999 range
  • whisper.cpp STT sidecar
  • 4 Whisper models (75MB–1.5GB)
  • 3-layer transcript formatting
  • Time capsules
  • SQLite WAL mode
  • Analytics bundle command
  • Structured logging
  • Wear OS voice capture
  • Rebrand → MoodHaven Journal
Zero cloud required. Two trusted devices on the same LAN sync within 30 seconds of appearing together.
Era 5 April 2026 v0.8.0

Openness

"No install required."

A full browser build backed by IndexedDB — the same zero-knowledge encryption model, any modern browser. Open it on a borrowed laptop. Close the tab. Nothing left behind. WebDAV sync uses ETag headers to prevent concurrent desktop/browser writes from clobbering each other.

  • Browser port (IndexedDB)
  • Identical zero-knowledge model
  • WebDAV sync for browser
  • ETag concurrency guard
  • PWA (installable)
  • npm run dev:web
  • VITE_TARGET=web build mode
  • isBrowser feature flag
Same encryption. Any browser. No account. No install. Nothing left on the machine after the tab closes.
Era 6 Apr–May 2026 v0.9.0–v1.0.0

Hardening

"From feature-complete to production-ready."

Password verification moved to Rust — the hash never leaves the backend. Seventeen Tauri commands gained lock guards. DOMPurify on all rendered journal HTML. Path traversal blocked in media commands. SHA-pinned CI supply chain. Rate limiting. 702 tests. Then v1.0.0 shipped.

  • verify_password in Rust
  • 17 commands lock-guarded
  • STT URL allowlist
  • DOMPurify integration
  • Path traversal blocking
  • Constant-time comparisons
  • Rate limiting (5 fail → 30s)
  • SHA-pinned CI actions
  • Vite 5→8 · vitest 1→4
  • Timeline virtual scroll
  • Keyboard shortcuts (1–5, ?)
  • STT recording strip UI
  • 702 tests · 48 files
  • v1.0.0 — stable release

Era 7 May 2026 v1.1.0–v1.3.0

Enrichment

"Settle the nervous system. Then write."

StillHaven brought bilateral audio stimulation — alternating left-right tones that help settle the nervous system before a journaling session. Bio-adaptive: reads Oura HRV at check-in, adjusts on live heart-rate signals from the Wear OS watch. Voice memo drafts made the watch a first-class writing tool.

  • StillHaven (bilateral audio)
  • Protocol picker + activation dial
  • Bio-adaptive (Oura + watch HRV)
  • Session history · journal handoff
  • Voice memo draft cards
  • Full-screen draft editor
  • Writing appearance drawer
  • TOTP encrypted at rest
  • Writer window scoped capability
  • Word count tracking
  • Session-linked journal entries
  • WellbeingCard (morning context)
  • Wrist Loop (watch → StillHaven)
  • 1,165 tests · 148 commands
StillHaven is the first feature that uses the journal as an outcome — not just a container.
